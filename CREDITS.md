# NFS Legacy Modpacks Credits

This document acknowledges the individuals, teams, and community projects whose work contributed to the creation of the **NFS Legacy Modpacks** project.

NFS Legacy Modpacks is first and foremost a **community preservation project**. The objective is not to replace the original games, but to preserve and enhance them by integrating carefully selected community-created improvements into a unified installer framework.

The installer architecture, validation workflow, rollback system, deployment pipeline, packaging, testing, integration, and release engineering were developed for NFS Legacy Modpacks. However, the enhancements included within each modpack are the work of many talented community members whose contributions deserve proper recognition.

---

# Attribution Philosophy

NFS Legacy Modpacks is built upon the work of the Need for Speed modding community.

Every included enhancement remains the intellectual property of its respective author(s). Whenever possible, original author names, aliases, teams, and project titles have been preserved exactly as they appear in their original releases.

This document acknowledges both:

* Modifications included in the released modpacks.
* Modifications evaluated during development but intentionally excluded after testing.

Documenting both categories preserves the project's engineering history and provides transparency regarding compatibility, stability, preservation goals, and technical decision-making throughout development.

# Naming Convention

Some contributors are listed using two names in the following format:

```text
Original Release Alias (Current / Common Alias)
```

This convention is used when an author has changed usernames over time or is commonly recognized by a different alias within the Need for Speed modding community.

The first name represents the alias used when the modification was originally published or distributed. The name in parentheses represents the author's current, updated, or more widely recognized alias where known.

Examples:

```text
ARCHIE (Archie)
nlgzrgn (nlgxzef)
_fliqpy_dem0n_ (fliqpydem0n)
```

This approach preserves historical accuracy while making it easier for users to identify authors, locate original releases, and recognize contributors whose public usernames have evolved over time.

Where only a single name is shown, no alternate alias was required or available at the time this document was prepared.

---

# About "Not Used"

Some entries include annotations such as:

```text
[Not Used – Could Break the Game]
[Not Used – No Longer Working]
[Not Used – Interferes With Other Mods]
```

These annotations indicate that a modification was evaluated during the development of NFS Legacy Modpacks but was intentionally excluded from the final Release 2.0 modpack.

Common reasons include:

* Compatibility conflicts
* Gameplay instability
* Engine limitations
* Broken functionality
* Conflicts with other selected modifications
* Preservation of the original gameplay experience

These annotations should **not** be interpreted as criticism of the original work. Instead, they document the engineering decisions made while assembling a stable, preservation-focused modpack.

Every listed project remains an important contribution to the Need for Speed modding community and is acknowledged here regardless of whether it was ultimately included.

Their appearance in this document should still be considered an acknowledgement of the author's work and its evaluation during development.

# Project Integration

While the individual modifications remain the work of their respective authors, the following components were developed specifically for **NFS Legacy Modpacks**:

* Release 2.0 installer architecture
* SetupLauncher
* LegacyUI
* Shared Inno Setup backend
* ArcRunner integration
* RestoreData rollback architecture
* Manifest-driven uninstall workflow
* SHA-256 validation methodology
* Release engineering workflow
* Documentation
* Repository organization
* Installer integration and compatibility testing
* Mod selection, validation, packaging, and quality assurance

The goal of the project is to provide a consistent, safe, and reproducible installation experience while preserving the identity of the original games and respecting the work of the community.
